> 이커머스 조직의 전체 역할 정의 + Claude 에이전트 파이프라인 매핑

---

## 2. 역할별 상세 책임

### 2.1 PM/PO

**핵심 미션:** 제품의 방향성을 설정하고 고객 가치를 극대화하는 의사결정을 내린다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| 시장 조사 | 경쟁사 분석 (무신사/29CM/W컨셉), 트렌드 리서치, 고객 인터뷰, 설문 조사 | 시장 분석 보고서, 경쟁사 벤치마크 |
| 방향성/목표 | 분기 OKR 설정, 제품 비전 정의, 우선순위 결정 | OKR 문서, 로드맵 |
| PRD | 기능 요구사항 정의, 유저 스토리 작성, 수용 조건(AC) 명세 | PRD 문서 |
| KPI | DAU, 전환율, ARPU, 재구매율, NPS, LTV, CAC 정의 및 추적 | KPI 대시보드, 주간 리포트 |
| A/B 테스트 | 실험 설계, 가설 수립, 표본 크기 산정, 결과 분석 | A/B 테스트 설계서, 실험 결과 보고서 |
| 릴리즈 관리 | Go/No-Go 판단, 점진적 롤아웃(1% → 10% → 50% → 100%), 피쳐 플래그 판단 | 릴리즈 체크리스트, 롤아웃 계획서 |
| 스테이크홀더 관리 | 경영진 보고, 부서 간 조율, 일정 커뮤니케이션 | 주간 보고, 이해관계자 맵 |

**필요 역량/도구:** Google Analytics, Amplitude, Mixpanel, Figma (뷰어), Jira, Confluence, SQL (기본), Retool

**에이전트 매핑:** `prd` 파이프라인이 PM/PO 역할을 수행. 시장 조사 에이전트가 경쟁사 분석을, PRD 작성 에이전트가 요구사항 명세를 담당.

---

### 2.2 디자이너

**핵심 미션:** 사용자 경험을 설계하고 일관된 디자인 시스템을 구축한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| UI/UX 설계 | 와이어프레임, 프로토타입, 유저 플로우, 인터랙션 설계 | Figma 디자인 파일 |
| 디자인 시스템 | 컴포넌트 라이브러리, 토큰 정의(색상/타이포/간격), 가이드라인 | 디자인 시스템 문서 |
| 접근성(a11y) | WCAG 2.1 AA 준수, 스크린 리더 호환, 키보드 내비게이션, 색상 대비 | 접근성 체크리스트 |
| 데이터 기반 디자인 | 히트맵 분석, 사용자 행동 분석, A/B 테스트 디자인 시안 | 사용성 분석 보고서 |

**필요 역량/도구:** Figma, Storybook, Zeplin, Hotjar, UserTesting

**에이전트 매핑:** 현재 Claude 에이전트가 직접 수행하지 않음. PM/PO 파이프라인에서 디자인 요구사항을 명세하고, FE 구현 시 디자인 시스템 준수 여부를 검증.

---

### 2.3 FE (프론트엔드)

**핵심 미션:** 사용자에게 직접 닿는 인터페이스를 빠르고 안정적으로 구현한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| Web/Mobile 구현 | React/Next.js 기반 컴포넌트 구현, 상태 관리, 라우팅 | 페이지/컴포넌트 코드 |
| API 통신 | REST/GraphQL 연동, 에러 핸들링, 로딩/재시도 전략 | API 연동 코드 |
| 성능 최적화 | LCP < 2.5s, FID < 100ms, CLS < 0.1, 번들 사이즈 최적화, 코드 스플리팅, 이미지 최적화(WebP/AVIF), 레이지 로딩 | 성능 리포트, Lighthouse 결과 |
| 에러 트래킹 | Sentry 연동, 에러 바운더리, 사용자 영향도 분류 | 에러 모니터링 대시보드 |

**필요 역량/도구:** React, Next.js, TypeScript, TanStack Query, Zustand/Recoil, Tailwind CSS, Sentry, Webpack/Vite, Lighthouse

**에이전트 매핑:** `implementation` 파이프라인의 FE 구현 에이전트가 수행. `prd` 파이프라인에서 FE 분석 에이전트(Agent A)가 프론트 영향도 분석.

---

### 2.4 BE (백엔드)

**핵심 미션:** 비즈니스 로직을 안전하고 확장 가능하게 구현하며, 데이터 정합성을 보장한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| DB 설계 | ERD, 정규화/반정규화, 인덱스 설계, FK/JSON/ENUM 미사용 원칙 | ERD, DDL, Flyway 마이그레이션 |
| 도메인 설계 | DDD (Aggregate, Entity, Value Object), 상태 머신, Saga Pattern | 도메인 모델 문서 |
| 비즈니스 로직 | 엔티티에 비즈니스 로직 캡슐화, Service는 얇게 유지, enum 상태 전이 규칙 | 도메인 코드 |
| API 구현/문서화 | REST API 설계, Swagger/OpenAPI 문서 자동화, 버전 관리 | API 엔드포인트, Swagger 문서 |
| 성능 | 슬로우 쿼리 EXPLAIN, N+1 방지, 캐싱 전략(Redis), 비동기 처리 | 쿼리 튜닝 결과 |
| 동시성 | 분산 락(Redisson), 낙관적/비관적 락, 멱등성 키 | 동시성 제어 코드 |
| 보안 | SQL Injection 방지, XSS/CSRF 방어, JWT 토큰 관리, 입력값 검증 | 보안 검증 코드 |

**필요 역량/도구:** Kotlin, Spring Boot 3.x, JPA/Hibernate, QueryDSL, Redis, Kafka, MySQL, Kotest, MockK, Testcontainers

**에이전트 매핑:** `be-implementation` 파이프라인이 BE 역할의 설계를, `implementation` 파이프라인이 구현을 수행. `prd` 파이프라인에서 BE 분석 에이전트(Agent B)가 백엔드 영향도 분석.

---

### 2.5 QA (품질 보증)

**핵심 미션:** 제품의 품질을 체계적으로 검증하고, 결함이 사용자에게 도달하지 않도록 차단한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| 피쳐 검증 | 수용 조건(AC) 기반 테스트, 경계값 분석, 동등 분할 | 테스트 케이스, 검증 리포트 |
| 핫픽스 검증 | 긴급 수정 사항 빠른 검증, 회귀 테스트 | 핫픽스 검증 리포트 |
| E2E 자동화 | Playwright/Selenium 기반 자동화 테스트, CI 통합 | E2E 테스트 코드, 실행 리포트 |
| 부하 테스트 | k6/nGrinder 활용, 타임세일 시나리오(1만 동시 주문), TPS 측정 | 부하 테스트 결과서 |
| 보안 테스트 | OWASP Top 10 점검, 취약점 스캔, 인증/인가 우회 테스트 | 보안 점검 보고서 |

**필요 역량/도구:** Playwright, Selenium, k6, nGrinder, Postman, OWASP ZAP, Jira (테스트 케이스 관리)

**에이전트 매핑:** `verification` 파이프라인이 QA 역할을 수행. 테스트 케이스 작성, 피쳐 검증, 검증 리포트 생성을 자동화.

---

### 2.6 DevOps

**핵심 미션:** 개발-배포-운영의 전체 사이클을 자동화하고, 안정적인 인프라를 제공한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| 환경 관리 | Docker Compose (로컬), EKS (프로덕션), dev/test/staging/prod 환경 분리 | 환경 설정 파일, Dockerfile |
| CI/CD | GitHub Actions 워크플로우, 블루-그린/카나리 배포 전략 | CI/CD 파이프라인, 배포 스크립트 |
| 인프라 관리 | Terraform IaC, AWS 리소스 관리(EKS, RDS, ElastiCache, S3) | Terraform 코드, 인프라 설계서 |
| 모니터링 | Grafana/Prometheus/Pinpoint/ELK 스택 구성 | 대시보드, 알림 룰 |
| 재해 복구 | RTO < 1h, RPO < 5min, 백업/복구 절차, 장애 시나리오별 대응 | DR 플레이북 |
| 비용 관리 | AWS 비용 최적화, Reserved Instance, Spot Instance 활용 | 비용 리포트 |
| SLO | 가용성 99.9%, p99 응답시간 < 500ms | SLO 대시보드 |

**필요 역량/도구:** Docker, Kubernetes, Terraform, AWS (EKS/RDS/S3/CloudFront), GitHub Actions, Grafana, Prometheus, Pinpoint, ELK

**에이전트 매핑:** `release` 파이프라인이 배포 관련 DevOps 역할을, `incident` 파이프라인이 장애 대응 역할을 수행.

---

### 2.7 테크리더

**핵심 미션:** 기술적 의사결정을 주도하고, 팀의 코드 품질과 아키텍처 일관성을 보장한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| 기술 선택 | 프레임워크, 라이브러리, 패턴 선택 및 PoC | 기술 검토 보고서 |
| ADR | Architecture Decision Record 작성, 의사결정 사유 기록 | ADR 문서 |
| 티켓 분할 | PRD → 기술 티켓 분해, 의존성 파악, 구현 순서 결정 | Jira 티켓, 의존성 그래프 |
| 일정 산출 | 티켓별 스토리 포인트, 스프린트 계획 | 스프린트 백로그 |
| 코드 리뷰 | 리뷰 가이드라인 수립, 머지 기준 정의, 아키텍처 준수 검증 | 코드 리뷰 가이드, 리뷰 코멘트 |
| 기술 부채 관리 | 스프린트 20% 기술 부채 해소 할당, 부채 목록 관리, 우선순위 결정 | 기술 부채 백로그 |

**필요 역량/도구:** 시스템 설계, DDD, 헥사고날 아키텍처, Jira, GitHub, draw.io/Mermaid

**에이전트 매핑:** `be-implementation` 파이프라인이 테크리더의 설계/분할 역할을, `pr-review` 파이프라인이 코드 리뷰 역할을, `refactoring` 파이프라인이 기술 부채 관리를 수행.

---

### 2.8 DBA (BE 겸임)

**핵심 미션:** 데이터베이스의 성능, 안정성, 데이터 정합성을 보장한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| 쿼리 튜닝 | EXPLAIN 분석, 인덱스 최적화, 쿼리 리팩토링 | 튜닝 결과 보고서 |
| 슬로우 쿼리 | 슬로우 쿼리 로그 모니터링, 임계치 알림 설정 | 슬로우 쿼리 대시보드 |
| 파티셔닝 | 대용량 테이블 파티셔닝 전략(Range/Hash), 아카이빙 | 파티셔닝 설계서 |
| 백업/복구 | 일간 풀백업, 시간별 증분 백업, Point-in-Time Recovery 검증 | 백업 절차서 |
| 마이그레이션 안전성 | Flyway 마이그레이션 리뷰, 무중단 DDL, 롤백 계획 | 마이그레이션 검증 리포트 |

**필요 역량/도구:** MySQL 8.0, EXPLAIN, pt-query-digest, Flyway, AWS RDS

**에이전트 매핑:** `prd` 파이프라인의 DB 분석 에이전트(Agent C)가 DB 영향도 분석. `be-implementation` 파이프라인에서 DB 설계 및 마이그레이션 검증 수행.

---

### 2.9 Data Engineer (BE/DevOps 겸임)

**핵심 미션:** 데이터를 수집, 가공, 분석하여 비즈니스 의사결정을 지원한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| 데이터 파이프라인 | Kafka → Data Lake → DW 파이프라인 구축, ETL/ELT | 파이프라인 코드, DAG |
| 매출/전환율 분석 | GMV, 전환 퍼널, 코호트 분석, 리텐션 분석 | 분석 대시보드 |
| BI | Metabase/Redash 기반 리포트, 자동화된 일간/주간 보고 | BI 대시보드, 정기 리포트 |

**필요 역량/도구:** Kafka, Spark/Flink, Airflow, Metabase, Redash, SQL, Python

**에이전트 매핑:** 현재 전용 파이프라인 없음. `prd` 파이프라인에서 데이터 분석 요구사항을 명세하고, `be-implementation` 파이프라인에서 데이터 파이프라인 구현을 설계.

---

### 2.10 SRE (DevOps 겸임)

**핵심 미션:** 서비스의 안정성과 신뢰성을 정량적으로 정의하고 지속적으로 개선한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| SLO/SLI 정의 | 가용성 SLO(99.9%), 지연 SLO(p99 < 500ms), 에러율 SLO(< 0.1%) | SLO 문서, 에러 버짓 |
| 장애 대응 | 온콜 체계, 에스컬레이션 절차, 장애 등급 분류(SEV1-4) | 장애 대응 런북 |
| 포스트모텀 | 근본 원인 분석(RCA), 재발 방지 액션 아이템, 타임라인 | 포스트모텀 문서 |
| 모니터링 | 인프라/애플리케이션/비즈니스 메트릭 모니터링, 알림 임계치 설정 | 모니터링 대시보드, 알림 룰 |

**필요 역량/도구:** Grafana, Prometheus, PagerDuty, Pinpoint, ELK, AWS CloudWatch

**에이전트 매핑:** `incident` 파이프라인이 장애 대응 및 포스트모텀 역할을 수행. `release` 파이프라인에서 배포 후 모니터링 체크리스트 생성.

---

### 2.11 보안 담당 (BE 겸임)

**핵심 미션:** 서비스와 데이터를 보호하고, 법적 컴플라이언스를 준수한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| PCI-DSS | 결제 데이터 처리 기준 준수, 카드 정보 비저장, 토큰화 | PCI-DSS 체크리스트 |
| 개인정보보호법 | 개인정보 수집/이용/파기 정책, 동의 관리, 열람/삭제 요청 처리 | 개인정보 처리방침, 영향평가서 |
| 암호화 | AES-256 (저장 시), TLS 1.3 (전송 시), BCrypt (비밀번호), 마스킹 | 암호화 정책 문서 |
| OWASP | OWASP Top 10 점검, 정기 취약점 스캔, 모의 해킹 | 보안 점검 보고서 |

**필요 역량/도구:** OWASP ZAP, SonarQube, Trivy, AWS WAF, KMS

**에이전트 매핑:** `pr-review` 파이프라인에서 보안 검증 에이전트가 코드의 보안 취약점을 점검. `verification` 파이프라인에서 OWASP 기반 보안 테스트 수행.

### 2.12 MD 머천다이저 (PM 겸임)

**핵심 미션:** 상품 기획과 판매 전략을 수립하여 매출을 극대화한다.

| 영역 | 상세 | 산출물 |
|------|------|--------|
| 브랜드 입점 | 입점 심사 기준, 브랜드 등급(프리미엄/일반), 계약 조건 | 입점 심사서, 계약서 |
| 기획전 | 시즌 기획전, 콜라보 기획전, 타임세일, 쿠폰 이벤트 | 기획전 기획서 |
| 가격 전략 | 가격 정책(정가/할인가/회원가), 마진율 관리, 경쟁사 가격 모니터링 | 가격 정책서 |
| 시즌 기획 | S/S, F/W 시즌 상품 기획, 트렌드 반영, 재고 계획 | 시즌 기획서, MD 캘린더 |

**필요 역량/도구:** Excel/Google Sheets, 데이터 분석, 트렌드 리서치, Retool (상품 관리)

**에이전트 매핑:** `prd` 파이프라인에서 MD 관련 요구사항을 분석. 기획전/프로모션은 PRD의 비즈니스 요구사항으로 반영.

## 3. 에이전트-파이프라인 매핑

| 파이프라인 | 대응 역할 | 에이전트 구성 | 산출물 |
|-----------|----------|--------------|--------|
| `prd` | PM/PO, MD | Agent A(FE 분석), Agent B(BE 분석), Agent C(DB 분석), Agent D(외부 연동 분석) | PRD 분석 보고서 |
| `be-implementation` | 테크리더, BE | 6개 에이전트 (도메인 설계, API 설계, DB 설계, 테스트 설계, 보안 검증, 티켓 분할) | TDD 설계서 + Jira 티켓 |
| `implementation` | BE, FE | 구현 에이전트 (코드 생성, 테스트 작성, 리팩토링) | 코드 + 테스트 + PR |
| `verification` | QA, BE | 검증 에이전트 (기능 검증, 회귀 테스트, 보안 점검) | 검증 리포트 |
| `inquiry` | BE, DevOps | Agent alpha(로그 분석), Agent beta(코드 추적) | 문의 대응 보고서 |
| `incident` | SRE, DevOps, BE | 3개 에이전트 (장애 감지, 원인 분석, 복구 실행) | 장애 대응 보고서 + 포스트모텀 |
| `pr-review` | 테크리더, 보안 | Agent 아키텍처(구조 검증), Agent 테스트(테스트 검증) | 코드 리뷰 보고서 |
| `release` | DevOps, SRE | Agent 배포(배포 실행), Agent 검증(배포 후 검증) | 배포 체크리스트 + 모니터링 리포트 |
| `refactoring` | 테크리더, BE | Agent 분석(코드 분석), Agent 설계(리팩토링 맵) | 리팩토링 계획서 |
| `api-change` | 테크리더, FE, BE | Agent API(변경 분석), Agent 호환(하위 호환성 검증) | API 변경 영향도 보고서 |

## 4. 개인 프로젝트 겸임 구조

1인 개발 + Claude 에이전트로 12개 역할을 수행하는 구조:

```
[나 (biuea)] ─── 의사결정, 방향성, 최종 승인
      │
      ▼
[Claude 메인 (테크리더)] ─── 오케스트레이션
      │
      ├─ PRD 파이프라인 ─────── PM/PO + MD
      ├─ BE 구현 설계 ────────── 테크리더 + BE
      ├─ 구현 파이프라인 ──────── BE + FE
      ├─ 검증 파이프라인 ──────── QA
      ├─ PR 리뷰 ──────────── 테크리더 + 보안
      ├─ 배포/릴리즈 ─────────── DevOps + SRE
      ├─ 장애 대응 ──────────── SRE
      └─ 문의 대응 ──────────── BE + DevOps
```

### 겸임 매핑

| 겸임 그룹 | 포함 역할 | 설명 |
|----------|----------|------|
| **BE** | BE + DBA + 보안 담당 | 백엔드 개발자가 DB 튜닝, 마이그레이션, 보안 검증까지 담당. 도메인 설계 시 DB 성능과 보안을 함께 고려. |
| **DevOps** | DevOps + SRE + Data Engineer | 인프라 담당이 배포, 모니터링, 장애 대응, 데이터 파이프라인까지 통합 관리. |
| **PM** | PM/PO + MD + 법무 | 제품 기획자가 머천다이징, 기획전, 가격 전략, 개인정보/전자상거래법 컴플라이언스까지 담당. |

### 의사결정 권한 분배

| 의사결정 유형 | biuea (사람) | Claude (에이전트) |
|-------------|-------------|------------------|
| 비즈니스 방향성 | 최종 결정 | 분석/제안 |
| 기술 선택 | 승인 | 조사/PoC/추천 |
| 코드 머지 | 최종 승인 | 리뷰/검증 |
| 배포 Go/No-Go | 최종 판단 | 체크리스트/리스크 분석 |
| 장애 대응 | 에스컬레이션 판단 | 초동 분석/복구 제안 |
| 아키텍처 변경 | 최종 승인 | ADR 작성/대안 분석 |

## 5. 역할 간 협업 플로우

일반적인 피쳐 개발 흐름에서 각 역할의 참여 순서:

```
PM/PO: 시장 조사 → PRD 작성
    ↓
디자이너: UI/UX 설계 (Figma)
    ↓
테크리더: PRD + 디자인 → TDD 작성 → 티켓 분할 → 일정 산출
    ↓
BE: DB 마이그레이션 → 도메인 구현 → API 구현 → 단위 테스트
FE: 컴포넌트 구현 → API 연동 → 통합 테스트
    ↓ (병렬)
QA: 테스트 케이스 작성 → 피쳐 검증 → E2E 자동화
    ↓
테크리더: 코드 리뷰 → 머지 승인
    ↓
DevOps: Staging 배포 → QA 최종 검증 → Production 배포
    ↓
SRE: 모니터링 → 알림 설정 → 포스트 배포 관찰
```

### Claude 에이전트 파이프라인 대응 흐름

```
[prd 파이프라인]         → PRD 분석 보고서
    ↓
[be-implementation]     → TDD 설계서 + Jira 티켓 생성
    ↓
[implementation]        → 코드 구현 + 테스트 작성
    ↓
[pr-review]             → 코드 리뷰 (아키텍처 + 보안)
    ↓
[verification]          → 검증 리포트
    ↓
[release]               → 배포 체크리스트 + 실행
    ↓
[모니터링]               → 배포 후 관찰 (incident 파이프라인 대기)
```

### 핫픽스/장애 대응 흐름

```
[incident 파이프라인]    → 장애 감지 + 원인 분석 + 복구 제안
    ↓
[biuea 승인]            → 복구 실행 판단
    ↓
[implementation]        → 핫픽스 코드 작성
    ↓
[verification]          → 핫픽스 검증
    ↓
[release]               → 긴급 배포
    ↓
[incident 파이프라인]    → 포스트모텀 작성
```

## 6. 이커머스 특화 체크리스트

### BE 이커머스 체크리스트

- [ ] 결제 금액 계산 정확성 (할인, 쿠폰, 포인트, 배송비)
- [ ] 재고 동시성 (분산 락 적용)
- [ ] 주문 상태 머신 전이 규칙
- [ ] PG 연동 멱등성 (결제 키 기반 중복 방지)
- [ ] 환불 금액 = 원래 결제 금액 (부분 환불 포함)
- [ ] 정산 금액 = 매출 - 수수료 - 환불
- [ ] 개인정보 암호화 (이름, 전화번호, 주소)
- [ ] Soft Delete 적용 (회원 탈퇴 시 30일 보관)

### QA 이커머스 체크리스트

- [ ] 주문 → 결제 → 배송 → 구매확정 Happy Path
- [ ] 주문 취소 + 재고 복원 + 환불
- [ ] 반품 신청 + 수거 + 검수 + 환불
- [ ] 품절 상품 주문 불가
- [ ] 쿠폰 중복 사용 불가
- [ ] 타임세일 종료 후 원가 복원
- [ ] 포인트 사용 후 취소 시 포인트 복원
- [ ] 동시 주문 재고 정합성 (부하 테스트)

### DevOps 이커머스 체크리스트

- [ ] PG 웹훅 엔드포인트 HTTPS only
- [ ] 결제 데이터 DB 암호화 (at rest)
- [ ] Kafka DLQ 설정 (주문/결제 이벤트 유실 방지)
- [ ] Redis 장애 시 Fallback (재고 조회)
- [ ] Elasticsearch 인덱스 재구축 배치
- [ ] DB 백업 주기 확인 (일간 풀백업 + 시간별 증분)

### PM 이커머스 체크리스트

- [ ] 전자상거래법 필수 표시 사항 (반품/교환/환불 정책)
- [ ] 통신판매업 신고 정보 표시
- [ ] 개인정보 수집/이용 동의 절차
- [ ] 결제 수단별 테스트 (카드/계좌이체/가상계좌/간편결제)
- [ ] 미성년자 구매 제한 (법정대리인 동의)
- [ ] 쿨링오프 기간 (7일) 환불 정책

### FE 이커머스 체크리스트

- [ ] 상품 목록 무한 스크롤/페이지네이션 성능
- [ ] 장바구니 → 주문 → 결제 플로우 이탈 추적
- [ ] 결제 중 새로고침/뒤로가기 처리
- [ ] 상품 이미지 레이지 로딩 + WebP 최적화
- [ ] 모바일 반응형 (상품 상세, 결제 페이지)
- [ ] SEO (상품 메타태그, 구조화 데이터, Open Graph)

### 보안 이커머스 체크리스트

- [ ] 카드 정보 비저장 (PG사 토큰 사용)
- [ ] 주소/전화번호 AES-256 암호화 저장
- [ ] 관리자 페이지 IP 화이트리스트
- [ ] API Rate Limiting (주문 API: 10req/s per user)
- [ ] 쿠폰/프로모션 코드 brute-force 방지
- [ ] 가격 변조 방지 (서버 사이드 가격 검증)
