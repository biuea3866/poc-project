## 3. 구현 설계

### 3.1 수정 파일 목록

```
[FE]
- path/to/component.tsx (수정 이유)

[BE]
- closet-{service}/module/path/to/File.kt (수정 이유)

[DB]
- closet-{service}/src/main/resources/db/migration/V{날짜}__{설명}.sql

[기타]
- Kafka 토픽 추가 시
- Elasticsearch 인덱스 매핑 변경 시
```

### 3.2 API 변경사항

```
[신규]
POST /api/v1/... - 설명

[수정]
PUT /api/v1/... - 변경 내용
```

### 3.3 데이터 흐름

```
사용자 액션 → FE 컴포넌트 → API 호출 → Gateway → BE Service → DB/Kafka
```

## 5. 구현 순서 (권장)

```
1단계: DB 마이그레이션 (있는 경우)
2단계: BE 도메인/서비스 구현
3단계: BE API 엔드포인트 추가
4단계: FE API 연동 및 UI 구현
5단계: Kafka 이벤트 연동 (있는 경우)
6단계: Elasticsearch 인덱스 동기화 (있는 경우)
7단계: 테스트 및 QA
```

## 6. 체크리스트

```
□ 하위 호환성 유지 여부
□ 기존 API 클라이언트 영향 없음
□ Kafka 메시지 포맷 호환
□ DB 마이그레이션 롤백 가능
□ 권한 체크 추가 필요 여부
□ 재고 정합성 영향 여부
□ 모니터링/로깅 추가 여부
```
