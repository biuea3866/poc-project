# 작업 파이프라인 — Closet E-Commerce

> 모든 작업은 이 파이프라인을 반드시 따른다. 단계를 건너뛰지 않는다.

---

## 전체 플로우

```
[사용자 지시]
    │
    ▼
━━ Phase 0: 기획 확인 ━━━━━━━━━━━━━━━━━━━━━━━━━━━
    │  PM: PRD 존재 여부 확인
    │  - 있으면 → Phase 1로
    │  - 없으면 → PRD 작성 먼저
    │
    ▼
━━ Phase 1: 설계 + 티켓 생성 ━━━━━━━━━━━━━━━━━━━━━
    │  TechLead (필수, 병렬 불가):
    │  1. PRD 읽고 기술 설계 (TDD)
    │  2. 후보군 비교 + 트레이드오프 (BE 필수)
    │  3. 티켓 분할 (하루~이틀 단위)
    │  4. Jira 에픽 + 작업 + 하위작업 생성
    │  5. Confluence 설계 문서 업데이트
    │
    │  산출물:
    │  - Jira 에픽 (CP-XX)
    │  - Jira 작업 티켓 (서비스별)
    │  - Jira 하위작업 (DB/도메인/API/테스트)
    │  - 기술 설계 문서 (.analysis/be-implementation/)
    │
    ▼
━━ Phase 2: 구현 (병렬) ━━━━━━━━━━━━━━━━━━━━━━━━━
    │  ⚠️ Jira 티켓이 없으면 구현 시작하지 않는다
    │
    │  BE (각 서비스별 worktree 병렬):
    │  1. Jira 티켓 상태 → "진행중"
    │  2. 브랜치 생성: feature/cp-{번호}-{기능}
    │  3. Flyway DDL → 도메인 → Service → Controller → 테스트
    │  4. 커밋: [CP-{번호}] {내용}
    │  5. 구현 기록 작성 (.analysis/be-implementation/results/)
    │
    │  FE (병렬):
    │  1. BFF API 기반 타입 정의
    │  2. 페이지/컴포넌트 구현
    │  3. API 연동
    │
    │  DevOps (병렬):
    │  1. Docker Compose 업데이트
    │  2. Gateway 라우팅 추가
    │  3. CI/CD 모듈 추가
    │  4. 모니터링 설정
    │
    ▼
━━ Phase 3: 검증 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    │  QA:
    │  1. 빌드 검증 (./gradlew compileKotlin)
    │  2. 테스트 실행 (./gradlew test)
    │  3. API TC 실행 (curl 기반)
    │  4. FE 통합 테스트
    │  5. 검증 리포트 작성 (.analysis/verification/results/)
    │
    │  TechLead:
    │  1. 코드 리뷰 (설계 원칙 준수)
    │  2. 기술 부채 식별
    │
    ▼
━━ Phase 4: 반영 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    │  1. PR 생성 (커밋 메시지에 [CP-{번호}] 포함)
    │  2. 머지
    │  3. 푸시
    │  4. Jira 티켓 상태 → "완료"
    │  5. Confluence 업데이트
    │  6. 구현 기록 최종화
    │
    ▼
━━ Phase 5: 지속 개선 (병렬, 항상 진행) ━━━━━━━━━━
    │  PM: 다음 Phase PRD 기획
    │  BE: 기술 부채 분석, 아키텍처 설계, 도메인 통합 설계
    │  FE: 성능 분석, UX 개선
    │  DevOps: 인프라 최적화, 모니터링 고도화
    │  QA: 테스트 커버리지 확대, 부하 테스트 시나리오 강화
```

## 역할별 책임 (각 Phase)

### Phase 0: 기획 확인

| 역할 | 책임 | 산출물 |
|------|------|--------|
| PM | PRD 존재 확인, 없으면 작성 | PRD 문서 |

### Phase 1: 설계 + 티켓 생성

| 역할 | 책임 | 산출물 |
|------|------|--------|
| TechLead | PRD → TDD, 후보군 비교, 티켓 분할 | TDD 문서, ADR |
| BE | 도메인 설계 리뷰, 트레이드오프 검토 | 설계 피드백 |
| Jira | 에픽 + 작업 + 하위작업 생성 | Jira 티켓 |

### Phase 2: 구현

| 역할 | 책임 | 산출물 |
|------|------|--------|
| BE | Jira 티켓 기반 구현 + 테스트 + 구현 기록 | 코드, 테스트, 구현 기록 |
| FE | BFF API 기반 화면 구현 | 페이지, 컴포넌트 |
| DevOps | 인프라 업데이트 | Docker, CI/CD, 모니터링 |

### Phase 3: 검증

| 역할 | 책임 | 산출물 |
|------|------|--------|
| QA | 빌드 + 테스트 + API TC + 검증 리포트 | QA 리포트 |
| TechLead | 코드 리뷰 + 기술 부채 식별 | 리뷰 코멘트 |

### Phase 4: 반영

| 역할 | 책임 | 산출물 |
|------|------|--------|
| TechLead | PR 생성 + 머지 | PR |
| 전체 | Jira 업데이트 + Confluence 업데이트 | 상태 변경 |

### Phase 5: 지속 개선 (항상)

| 역할 | 책임 | 산출물 |
|------|------|--------|
| PM | 다음 Phase PRD + 경쟁사 분석 + KPI | PRD, 분석 보고서 |
| BE | 기술 부채 + 아키텍처 설계 + 도메인 통합 (후보군+트레이드오프) | 설계 문서, ADR |
| FE | Core Web Vitals + 번들 분석 + UX 개선 | 분석 보고서 |
| DevOps | 인프라 비용 + 배포 전략 + 모니터링 | 개선 보고서 |
| QA | 테스트 커버리지 + E2E + 부하 테스트 | 테스트 전략 |

## 금지 사항

| # | 금지 | 이유 |
|---|------|------|
| 1 | **Jira 티켓 없이 구현 시작** | 작업 추적 불가 |
| 2 | **main 직접 커밋/푸시** | 반드시 feature 브랜치 + PR |
| 3 | **구현 기록 없는 PR** | 리뷰 거부 사유 |
| 4 | **설계 문서 없이 대규모 변경** | 후보군 비교 + 트레이드오프 필수 |
| 5 | **QA 검증 없이 머지** | 빌드 + 테스트 통과 필수 |
| 6 | **Confluence 미업데이트** | 모든 산출물은 .md + Confluence 양쪽 |
| 7 | **Client가 BFF 거치지 않고 직접 서비스 호출** | Client → Gateway → BFF만 허용 |

## "다음 로드맵 해줘" 실행 시 에이전트 동작

```
1. PM 에이전트: PRD 확인 → 없으면 작성
2. TechLead 에이전트: TDD 작성 + Jira 티켓 생성
3. [여기까지 완료 후]
4. BE 에이전트 N개 (worktree 병렬): 구현
5. FE 에이전트: BFF API 연동
6. DevOps 에이전트: 인프라 업데이트
7. [구현 완료 후]
8. QA 에이전트: 검증
9. TechLead 에이전트: 리뷰 + 머지 + Jira/Confluence 업데이트
10. [병렬] PM: 다다음 Phase PRD, BE: 기술 부채 분석
```

## 체크리스트

작업 시작 전:
- [ ] PRD가 존재하는가?
- [ ] Jira 에픽이 생성되었는가?
- [ ] Jira 작업 티켓이 생성되었는가?
- [ ] 기술 설계 문서(TDD)가 작성되었는가?

구현 중:
- [ ] feature 브랜치에서 작업하고 있는가?
- [ ] 커밋 메시지에 [CP-{번호}]가 포함되어 있는가?
- [ ] 구현 기록을 .analysis/에 작성하고 있는가?

구현 완료 후:
- [ ] 빌드가 성공하는가?
- [ ] 테스트가 전부 통과하는가?
- [ ] QA 검증 리포트가 작성되었는가?
- [ ] PR이 생성되었는가?
- [ ] Jira 상태가 "완료"로 변경되었는가?
- [ ] Confluence가 업데이트되었는가?
