## 1. Phase 1 KPI (핵심 커머스)

| KPI | 정의 | 목표 | 측정 방법 | 주기 |
|-----|------|------|----------|------|
| **회원가입 전환율** | 방문자 중 가입한 비율 | 5% | `member` 테이블 신규 가입 수 / GA 방문자 수 | 일간 |
| **주문 전환율** | 상품 조회 → 주문 완료 비율 | 2% | `order` 완료 건수 / 상품 조회 수 (GA 이벤트) | 일간 |
| **주문 완료율** | 주문 시작 → 결제 완료 비율 | 70% | status=`PAID` 건수 / status=`PENDING` 이상 건수 | 일간 |
| **평균 주문 금액 (AOV)** | 주문 건당 평균 결제 금액 | 50,000원 | `SUM(payment_amount) / COUNT(DISTINCT order_id)` | 일간 |
| **결제 성공률** | 결제 시도 → PG 승인 비율 | 95% | status=`APPROVED` / 전체 결제 시도 건수 | 실시간 |
| **API 응답 시간 (p95)** | 95번째 백분위 응답 시간 | < 200ms | Prometheus `http_server_requests_seconds` | 실시간 |

### Phase 1 보조 지표

| KPI | 정의 | 목표 | 측정 방법 |
|-----|------|------|----------|
| DAU | 일일 활성 사용자 수 | 1,000명 | GA |
| WAU | 주간 활성 사용자 수 | 5,000명 | GA |
| MAU | 월간 활성 사용자 수 | 15,000명 | GA |
| 상품 등록 수 | 누적 상품 수 | 500개 | `product` 테이블 COUNT |
| 주문 건수 | 일일 주문 건수 | 50건/일 | `order` 테이블 일별 COUNT |
| 에러율 | 5xx 에러 비율 | < 0.1% | Prometheus |
| 서버 가용성 | 서비스 업타임 | 99.9% | Health Check |

## 2. Phase 2 KPI (성장)

| KPI | 정의 | 목표 | 측정 방법 | 주기 |
|-----|------|------|----------|------|
| **검색 사용률** | 검색을 사용한 세션 비율 | 30% | 검색 API 호출 세션 / 전체 세션 | 일간 |
| **검색 전환율** | 검색 → 구매 전환 비율 | 5% | 검색 후 주문 완료 건수 / 검색 세션 수 | 일간 |
| **리뷰 작성률** | 구매확정 → 리뷰 작성 비율 | 20% | `review` 건수 / `CONFIRMED` 주문 건수 | 주간 |
| **반품률** | 배송 완료 → 반품 비율 | < 10% | `return_request` 건수 / `DELIVERED` 주문 건수 | 주간 |
| **재구매율** | 30일 내 재구매 비율 | 15% | 30일 내 2회 이상 주문 회원 / 전체 주문 회원 | 월간 |

### Phase 2 보조 지표

| KPI | 정의 | 목표 | 측정 방법 |
|-----|------|------|----------|
| 검색 클릭률 (CTR) | 검색 결과 클릭 비율 | 40% | 검색 결과 클릭 / 검색 노출 |
| 검색 결과 없음 비율 | 검색 결과가 0건인 비율 | < 5% | 0건 결과 검색 / 전체 검색 |
| 평균 리뷰 별점 | 전체 리뷰 평균 별점 | 4.0+ | `AVG(rating)` from `review` |
| 포토 리뷰 비율 | 포토 리뷰 / 전체 리뷰 | 30% | `has_photo=1` / 전체 리뷰 |
| 사이즈 리뷰 비율 | 사이즈 정보 포함 리뷰 비율 | 40% | `review_size_info` 건수 / 전체 리뷰 |
| 교환 처리 시간 | 교환 신청 → 완료 평균 시간 | < 5일 | `completed_at - requested_at` 평균 |
| 배송 소요 시간 | 발송 → 배송 완료 평균 시간 | < 2일 | `delivered_at - shipped_at` 평균 |
| 재고 품절률 | 전체 SKU 중 품절 비율 | < 5% | `quantity=0` SKU / 전체 SKU |
| 재입고 알림 전환율 | 재입고 알림 → 구매 비율 | 20% | 알림 후 구매 건 / 알림 발송 건 |
| 자동완성 사용률 | 자동완성 클릭 / 전체 검색 | 25% | 자동완성 클릭 이벤트 / 검색 이벤트 |

## 3. KPI 측정 인프라

### 데이터 수집 아키텍처

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  Application │     │    Kafka     │     │   Metrics    │
│   Services   │────>│  (이벤트)    │────>│   Consumer   │
└──────────────┘     └──────────────┘     └──────┬───────┘
       │                                         │
       │ Prometheus                              │
       ▼                                         ▼
┌──────────────┐                         ┌──────────────┐
│  Prometheus  │                         │   Analytics  │
│  + Grafana   │                         │     DB       │
└──────────────┘                         └──────┬───────┘
       │                                         │
       └─────────────────┬───────────────────────┘
                         ▼
                 ┌──────────────┐
                 │   Grafana    │
                 │  Dashboard   │
                 └──────────────┘
```

### 대시보드 구성

#### 실시간 대시보드 (Grafana)
- **서비스 헬스**: API 응답 시간, 에러율, TPS
- **비즈니스**: 실시간 주문 건수, 결제 금액, 검색 쿼리 수
- **인프라**: CPU/메모리, DB 커넥션, Redis 히트율

#### 일간 리포트 (자동 발송)
- 전일 주요 KPI 요약
- 전주 대비 변동률
- 이상 징후 알림 (KPI 목표 미달 시)

#### 주간 리포트
- 주간 트렌드 분석
- 퍼널 분석 (방문 → 검색 → 조회 → 장바구니 → 주문 → 결제)
- Top 10 검색어, Top 10 상품

## 4. 알림 규칙

| 구분 | 조건 | 채널 | 우선순위 |
|------|------|------|---------|
| **결제 성공률 하락** | < 90% (5분간) | Slack #alert-critical | P1 |
| **API 응답 시간 증가** | p95 > 500ms (5분간) | Slack #alert-warning | P2 |
| **에러율 증가** | 5xx > 1% (5분간) | Slack #alert-critical | P1 |
| **주문 건수 급감** | 전일 동시간 대비 50% 감소 | Slack #alert-warning | P2 |
| **검색 결과 없음 급증** | > 20% (1시간) | Slack #alert-info | P3 |
| **반품률 증가** | > 15% (주간) | Slack #alert-warning | P2 |
| **재고 품절 급증** | 품절 SKU > 10% | Slack #alert-warning | P2 |
