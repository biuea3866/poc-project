# RAG Search UX Spec

> 통합 검색 페이지에 RAG 벡터 유사도 검색 탭을 추가하는 UI/UX 스펙

---

## 1. 현재 검색 UX 분석

### 현재 상태 (MVP)

- **경로:** `/search`
- **기능:** 검색 입력 필드만 존재, API 미연동
- **검색 방식:** `GET /api/v1/search/integrated` (LIKE 검색) — 제목/내용 키워드 매칭
- **문제점:**
  - 검색 결과 렌더링 미구현
  - 유사도 기반 검색 없음 — 정확한 키워드가 아니면 결과 누락
  - 웹 검색 탭 미구현

### 개선 방향

1. 키워드 검색(LIKE) 결과 섹션 구현
2. RAG 벡터 유사도 검색 탭 추가
3. 웹 검색 탭 추가 (별도 영역)

---

## 2. 검색 페이지 레이아웃

### 전체 구조

```
┌──────────────────────────────────────────────────────────────┐
│  검색                                                        │
│  키워드와 AI 기반 통합 검색 결과를 확인하세요.                     │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ [🔍] 문서 제목, 내용, 태그 검색...              [Enter] │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                     │
│  │ 키워드(3) │ │ AI 추천(5)│ │ 웹 검색   │  ← 탭 네비게이션   │
│  └──────────┘ └──────────┘ └──────────┘                     │
│  ─────────────────────────────────────────────────────────── │
│                                                              │
│  [검색 결과 영역 — 선택된 탭에 따라 변경]                       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 탭 구성

| 탭 | API | 설명 |
|---|---|---|
| **키워드** | `GET /api/v1/search/integrated?q=&page=&size=` | LIKE 검색 (제목/내용) |
| **AI 추천** | `GET /api/v1/search/rag?q=&page=&size=` | 벡터 유사도 검색 |
| **웹 검색** | `GET /api/v1/search/web?q=` | 외부 웹 검색 |

### 탭 스타일

```
활성 탭:    font-semibold text-accent border-b-2 border-accent
비활성 탭:  text-muted hover:text-secondary
탭 카운트:  ml-1.5 text-xs bg-accent-light text-accent rounded-full px-1.5
```

## 3. 검색 입력 UX

### 입력 → 검색 흐름

```
[사용자 입력] → 300ms debounce → 키워드 탭 자동 검색
                                → AI 추천 탭은 Enter 또는 탭 클릭 시 검색 (비용 고려)
```

### 검색 입력 필드

- 기존 스타일 유지: `rounded-xl border py-4 pl-12 pr-4 text-lg`
- Enter 키: 모든 탭 결과 갱신
- 검색어 비우기: 입력 필드 우측 `X` 버튼 (검색어 존재 시에만 노출)
- 최소 검색 글자 수: 2자

## 4. 키워드 검색 결과 카드

### 레이아웃

```
┌────────────────────────────────────────────────────────┐
│  문서 제목                                              │
│  본문 일부... 검색어가 **하이라이트** 되어 표시...        │  ← line-clamp-2
│                                                        │
│  [# Kotlin]  [# Spring]     2024-03-10  ·  작성자      │
└────────────────────────────────────────────────────────┘
```

### 하이라이트 규칙

- 검색 키워드와 매칭되는 텍스트: `bg-warning/20 text-primary font-semibold`
- 제목과 본문 모두에 적용

## 5. RAG (AI 추천) 검색 결과 카드

### 레이아웃

```
┌────────────────────────────────────────────────────────┐
│  ┌────────┐                                            │
│  │AI 추천 │  유사도 92%                                 │
│  └────────┘                                            │
│                                                        │
│  문서 제목                                              │
│  임베딩 유사도 기반으로 매칭된 본문 조각(chunk)이          │
│  여기에 표시됩니다. 검색어와 **관련된 부분**이 강조...     │  ← line-clamp-3
│                                                        │
│  [# 태그1]  [# 태그2]     2024-03-10                    │
└────────────────────────────────────────────────────────┘
```

### 상세 스펙

| 요소 | 스타일 | 설명 |
|---|---|---|
| AI 추천 뱃지 | `Badge variant="info"` — `bg-accent-light text-accent text-xs font-semibold rounded-full px-2.5 py-0.5` | 벡터 검색 결과 표시 |
| 유사도 점수 | `text-sm font-bold text-accent` | 0~100% 표시 (서버에서 0~1로 반환, FE에서 × 100) |
| 유사도 바 | `h-1 rounded-full bg-accent-light` 안에 `bg-accent` 너비 비율 | 시각적 스코어 표현 |
| 문서 제목 | `text-lg font-bold text-primary` | 클릭 시 `/documents/{id}`로 이동 |
| Chunk 본문 | `text-sm text-secondary line-clamp-3` | chunk_content에서 발췌 |
| 하이라이트 | `bg-accent/10 text-accent font-medium` | 유사 키워드 강조 |
| 태그 | `Badge variant="tag"` | 문서에 할당된 태그 |

### 결과 정렬

- 기본: 유사도 점수 내림차순 (가장 관련 높은 문서가 최상단)
- 결과 간 간격: `space-y-4`

## 8. API Contract

### `GET /api/v1/search/rag`

#### Request

| Parameter | Type | Required | 설명 |
|---|---|---|---|
| `q` | string | Yes | 검색 쿼리 (최소 2자) |
| `page` | number | No (default: 0) | 페이지 번호 |
| `size` | number | No (default: 10) | 페이지 크기 |

#### Response

```json
{
  "content": [
    {
      "documentId": 42,
      "title": "Kotlin Coroutines 가이드",
      "chunkContent": "코루틴은 비동기 프로그래밍을 위한...",
      "similarityScore": 0.92,
      "tags": [
        { "id": 1, "name": "Kotlin", "tagType": "TECH_STACK" },
        { "id": 5, "name": "비동기", "tagType": "TOPIC" }
      ],
      "createdAt": "2024-03-10T14:30:00",
      "updatedAt": "2024-03-12T09:15:00"
    }
  ],
  "totalElements": 23,
  "totalPages": 3,
  "number": 0,
  "size": 10
}
```

#### FE에서 필요한 데이터 매핑

| 서버 필드 | UI 요소 | 변환 |
|---|---|---|
| `documentId` | 카드 클릭 링크 | `/documents/${documentId}` |
| `title` | 카드 제목 | 그대로 표시 |
| `chunkContent` | 본문 발췌 | `line-clamp-3`, 키워드 하이라이트 |
| `similarityScore` | 유사도 표시 | `Math.round(score * 100) + "%"` |
| `tags` | 태그 뱃지 목록 | `Badge variant="tag"` |
| `createdAt` | 메타 정보 | `toLocaleDateString()` |
| `totalElements` | 탭 카운트 | 탭 이름 옆 `(n)` |

## 10. 반응형 고려사항

| Breakpoint | 변경 사항 |
|---|---|
| `md` 이상 (768px+) | 기본 레이아웃 — 카드 전체 너비 |
| `sm` 미만 (640px-) | 탭 가로 스크롤, 카드 패딩 축소 (`p-4`), 유사도 바 숨김 |
