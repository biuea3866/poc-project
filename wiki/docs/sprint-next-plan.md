## 4. 디자인 핸드오프 요청 (NAW-110)

> 디자이너에게 전달할 UI/UX 요구사항

### 4-1. 시맨틱 검색 결과 UI

**요구사항:**
- 기존 검색 페이지에 탭 추가: `키워드` | `시맨틱` | `하이브리드`
- 시맨틱 검색 결과 카드 디자인:
  - 문서 제목
  - 매칭된 텍스트 snippet (키워드 하이라이트)
  - 유사도 점수 시각화 (프로그레스 바 또는 백분율 뱃지)
  - 태그 뱃지 목록
- 하이브리드 모드: 키워드 매칭 결과와 시맨틱 매칭 결과를 통합 랭킹으로 표시
- 빈 결과 상태: "관련 문서를 찾지 못했습니다" + 검색어 제안

**참고 레퍼런스:**
- Notion의 검색 결과 UI (깔끔한 카드 리스트)
- Perplexity의 시맨틱 검색 결과 표시 (관련도 점수 포함)

### 4-2. 어드민 Outbox 모니터링 UI (선택)

**요구사항:**
- 실패한 이벤트 목록 (테이블 형태)
- 상태 필터: PENDING / SUCCESS / DEAD_LETTER
- 수동 재처리 버튼
- 이벤트 상세 모달 (payload JSON 뷰어)

> 어드민 UI는 MVP 이후 별도 스프린트에서 구현 가능. 이번 스프린트에서는 API만 구현하고 Swagger로 대체해도 무방.

## 5. 리스크 및 완화 방안

| 리스크 | 영향 | 완화 방안 |
|---|---|---|
| pgvector 검색 성능이 대규모 데이터에서 저하 | 검색 응답 시간 증가 | HNSW 인덱스 파라미터 튜닝, ef_search 조정. 초기에는 문서 수가 적으므로 성능 이슈 가능성 낮음 |
| Outbox 스케줄러가 대량 미처리 이벤트 발생 시 부하 | Kafka 재발행 폭주 | batch size 제한 (최대 100건/주기), rate limiting 적용 |
| Refresh 토큰 회전 도입 시 기존 세션 강제 로그아웃 | 사용자 경험 저하 | 마이그레이션 기간 동안 기존 토큰도 유효하게 처리 (grace period 1일) |
| 디자인 핸드오프 지연 | FE 개발 착수 지연 | FE는 Week 1에 기존 검색 UI 리팩토링 먼저 진행, 디자인 확정 후 스타일 적용 |

## 6. 성공 기준

- [ ] Kafka 발행 실패 시 outbox를 통해 100% 복구 가능
- [ ] 시맨틱 검색으로 키워드가 다른 유사 문서를 찾을 수 있음
- [ ] Refresh 토큰 탈취 시 같은 family의 모든 토큰이 무효화됨
- [ ] Grafana에 SLO 알람이 설정되어 장애 시 즉시 알림 수신
- [ ] 모든 신규 API에 통합 테스트 존재
