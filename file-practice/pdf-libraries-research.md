# JVM PDF 라이브러리 조사 결과

## 요구사항 분석
- JVM 환경에서 PDF 파일을 생성하고 다운로드하는 기능이 필요합니다.
- 주문 내역 리스트를 PDF로 다운로드할 수 있어야 합니다.
- 주문 내역은 상품, 쿠폰, 주문 정보로 구성되어 있습니다.

## 후보 라이브러리

### 1. iText
- **개요**: PDF 생성 및 조작을 위한 강력하고 기능이 풍부한 라이브러리입니다.
- **장점**:
    - PDF 생성, 조작, 추출 등 광범위한 기능을 제공합니다.
    - 텍스트 포맷팅, 테이블, 이미지, 암호화, 디지털 서명 등 고급 기능을 지원합니다.
    - 활발한 커뮤니티와 좋은 문서화를 가지고 있습니다.
- **단점**:
    - 대부분의 상업적 용도에는 상업용 라이선스가 필요합니다. (AGPL 또는 상업용 라이선스)

### 2. Apache PDFBox
- **개요**: Apache 재단에서 개발한 오픈소스 라이브러리로, PDF 문서 생성, 조작, 내용 추출 기능을 제공합니다.
- **장점**:
    - 완전한 오픈소스이며 무료로 사용할 수 있습니다.
    - PDF 문서 생성, 기존 문서 조작, 텍스트 및 이미지 추출 등 기본적인 PDF 처리 기능을 제공합니다.
    - 암호화, 디지털 서명, 폼 필링 등의 기능도 포함합니다.
- **단점**:
    - iText에 비해 API가 다소 저수준(low-level)이어서 학습 곡선이 가파를 수 있습니다.
    - 복잡한 레이아웃이나 고급 디자인을 구현하기 위해서는 더 많은 코드가 필요할 수 있습니다.
    - 대규모 작업 시 리소스 집약적일 수 있습니다.

### 3. IronPDF
- **개요**: 상업용 Java PDF 라이브러리로, 특히 HTML-to-PDF 변환에 강점을 가집니다.
- **장점**:
    - HTML, CSS, JavaScript를 사용하여 웹 페이지를 정확하게 PDF로 변환할 수 있습니다.
    - 서명, 워터마킹, 텍스트 및 이미지 추출 기능도 제공합니다.
- **단점**:
    - 상업용 라이브러리이므로 비용이 발생합니다.
    - HTML 기반이 아닌 순수 프로그래밍 방식의 PDF 생성에는 적합하지 않을 수 있습니다.

### 4. Flying Saucer
- **개요**: 오픈소스 라이브러리로, HTML 또는 XHTML 파일을 PDF로 변환하는 데 특화되어 있습니다.
- **장점**:
    - CSS 스타일링을 잘 지원하여 웹 콘텐츠를 PDF로 변환하는 데 유용합니다.
- **단점**:
    - 주로 HTML 변환에 중점을 두므로, 처음부터 프로그래밍 방식으로 PDF를 생성하는 데는 제한적일 수 있습니다.

## 결론
- **iText**는 강력하지만 상업용 라이선스 문제로 인해 오픈소스 프로젝트에는 부담이 될 수 있습니다.
- **IronPDF**와 **Flying Saucer**는 HTML-to-PDF 변환에 특화되어 있어, 현재 요구사항(주문 내역 데이터를 기반으로 PDF 생성)에는 직접적으로 맞지 않을 수 있습니다.
- **Apache PDFBox**는 완전한 오픈소스이며, PDF 문서 생성 및 조작에 필요한 기본적인 기능을 제공합니다. API가 다소 저수준일 수 있지만, 프로그래밍 방식으로 데이터를 PDF에 직접 그리는 데 필요한 유연성을 제공합니다.

**최종 선택: Apache PDFBox**

- 이유: 오픈소스이며, 주문 내역 데이터를 기반으로 PDF를 생성하는 데 필요한 핵심 기능을 제공합니다. 복잡한 레이아웃보다는 데이터 표현에 중점을 둔 PDF 생성에 적합합니다.
