# 배포 영향 분석 (Release Impact) 파이프라인

> 배포 전 이번 릴리즈에 포함된 변경들의 전체 영향 범위 사전 확인

---

## 파이프라인 흐름

```
[INPUT] 배포 대상 서비스 + 브랜치 (또는 PR 목록)
    │
    ▼
━━ Phase 1: 변경 사항 수집 ━━━━━━━━━━━━━━━━━━━━━━━━
    │  팀장이 배포 범위 파악
    │  - git log (dev vs main/master) 로 변경 커밋 수집
    │  - 변경 파일 전체 목록
    │  - 포함된 PR/티켓 목록
    │
    ▼
━━ Phase 2: 영향 분석 (병렬) ━━━━━━━━━━━━━━━━━━━━━━
    │  ┌─ Agent 🚀: 서비스 간 영향 분석
    │  └─ Agent ⚡: 위험 변경 탐지
    │
    ▼
━━ Phase 3: 배포 체크리스트 생성 ━━━━━━━━━━━━━━━━━━━
    │  - 배포 순서 결정
    │  - 사전/사후 작업 목록
    │  - 롤백 계획
    │  - QA 체크 항목
    │
    ▼
[OUTPUT] 배포 영향 분석 보고서
```

---

## 에이전트 역할 정의

### 팀장 (메인 Claude)
- git log/diff로 변경 커밋 수집
- 에이전트 결과 종합
- 최종 배포 체크리스트 작성
- 배포 순서 결정

### Agent 🚀: 서비스 간 영향 분석가
**목표**: 이번 배포가 다른 서비스에 미치는 영향 확인

**분석 항목**:
1. API 엔드포인트 변경 → 호출하는 다른 서비스/FE 확인
2. Kafka 메시지 포맷 변경 → Consumer 호환성 확인
3. DB 스키마 변경 → 다른 서비스의 Entity 호환성
4. 공유 라이브러리 버전 변경 → 소비 서비스 호환성
5. 설정 변경 → 환경변수 추가 필요 여부 (DevOps 전달)

**크로스 서비스 의존성 체크**:
```
변경 서비스 → 이 서비스를 호출하는 서비스 목록
    ↓
각 호출 서비스에서 사용하는 API/이벤트 확인
    ↓
변경된 인터페이스와 호환되는지 확인
    ↓
함께 배포해야 하는 서비스 식별
```

### Agent ⚡: 위험 변경 탐지가
**목표**: 배포 시 장애를 유발할 수 있는 위험 변경 탐지

**탐지 항목**:
1. **DB 마이그레이션 포함 여부**
   - 컬럼 삭제/타입 변경 → 롤백 불가 위험
   - 대용량 테이블 ALTER → 잠금 위험
   - 인덱스 추가 → 느린 마이그레이션 가능

2. **하위 호환성 깨짐**
   - API 응답 필드 삭제/이름 변경
   - 필수 파라미터 추가 (기존 클라이언트 깨짐)
   - Kafka 메시지 필드 변경

3. **설정 의존성**
   - 새 환경변수 추가 (인프라 사전 배포 필요)
   - 새 Kafka 토픽 (Terraform 사전 적용 필요)
   - 새 DB 테이블 (마이그레이션 사전 실행 필요)

4. **동시 배포 필수 여부**
   - FE + BE 동시 배포 필요한 경우
   - 여러 BE 서비스 동시 배포 필요한 경우

## 배포 순서 결정 규칙

```
1단계: 인프라 (Terraform - Kafka 토픽, 환경변수)
    ↓
2단계: DB 마이그레이션 (Flyway)
    ↓
3단계: 공유 라이브러리 (doodlin-commons, spring-kafka)
    ↓
4단계: BE 서비스 (의존되는 서비스부터)
    순서: authn/authz → new-back/ats → communication → aggregator → gateway
    ↓
5단계: FE 서비스
    ↓
6단계: Worker/Batch 서비스
```

## 에이전트 할당 규칙

| 배포 유형 | 할당 에이전트 |
|----------|-------------|
| 단일 FE 앱 배포 | 에이전트 없이 팀장만 |
| 단일 BE 서비스 (API 변경 없음) | 에이전트 없이 팀장만 |
| 단일 BE 서비스 (API 변경 있음) | 🚀만 |
| BE 다중 서비스 배포 | 🚀 + ⚡ |
| DB 마이그레이션 포함 | ⚡만 (위험 분석 집중) |
| 공유 라이브러리 배포 | 🚀 + ⚡ |
| 전체 스택 배포 | 🚀 + ⚡ |

## 배포 체크리스트 템플릿

```
□ 사전 작업
  □ 환경변수 추가 확인 (DevOps)
  □ Kafka 토픽 생성 확인
  □ DB 마이그레이션 실행
  □ 공유 라이브러리 배포

□ 배포
  □ BE 서비스 배포 순서 확인
  □ FE 앱 배포
  □ Worker 배포

□ 사후 확인
  □ 헬스체크 정상
  □ 에러 로그 모니터링 (5분)
  □ 핵심 기능 스모크 테스트
  □ Kafka Consumer lag 확인

□ 롤백 계획
  □ 롤백 트리거 조건 명확
  □ 롤백 순서 (배포 역순)
  □ DB 롤백 가능 여부
```

## 출력 어조

산출물은 **팀원과 공유하는 문서**입니다. 읽는 사람이 지치지 않도록 아래 원칙을 따릅니다.

- **핵심부터** — 결론·액션 아이템을 앞에 씁니다. 이유와 배경은 뒤에 써도 됩니다.
- **짧게** — 한 문장으로 쓸 수 있으면 세 문장으로 쓰지 않습니다.
- **구체적으로** — "여러 곳" 대신 "3곳", "느릴 수 있음" 대신 "users 테이블 full scan" 처럼 씁니다.
- **표·불릿 우선** — 비교·목록은 문장보다 표나 불릿으로 씁니다.
- **중립적으로** — 문제를 발견해도 단정짓지 않고 확인 사항으로 전달합니다.
