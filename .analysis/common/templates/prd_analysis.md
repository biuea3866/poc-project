## 3. 구현 설계

### 3.1 수정 파일 목록

```
[FE]
- repo/path/to/file.tsx (수정 이유)

[BE]
- repo/module/path/to/File.kt (수정 이유)

[DB]
- greeting-db-schema/V{날짜}__{설명}.sql

[기타]
- greeting-topic/ (토픽 추가 시)
```

### 3.2 API 변경사항

```
[신규]
POST /service/ats/api/v1.0/... - 설명

[수정]
PUT /service/ats/api/v1.0/... - 변경 내용
```

### 3.3 데이터 흐름

```
사용자 액션 → FE 컴포넌트 → API 호출 → Gateway → BE Service → DB/Kafka
```

## 6. 체크리스트

```
□ 하위 호환성 유지 여부
□ 기존 API 클라이언트 영향 없음
□ Kafka 메시지 포맷 호환
□ DB 마이그레이션 롤백 가능
□ 권한 체크 추가 필요 여부
□ 국제화 키 추가 여부
□ 모니터링/로깅 추가 여부
```
