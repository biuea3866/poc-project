# Stage 03: 릴리즈 로드맵

> PRD v2.1 기준 | 작성일: 2026-03-14 | 갱신일: 2026-03-14 (신규 기능 5건 추가)

---

## 로드맵 개요

AI Wiki를 5개 마일스톤으로 나누어 점진적으로 릴리즈한다.
M1~M3은 MVP 범위, M4~M5는 Post-MVP 확장 기능이다.
각 마일스톤은 독립적으로 검증 가능한 단위이며, 마일스톤 내 작업은 FE/BE/DevOps lane 병렬로 진행한다.

## 마일스톤 의존성

```
M1 ──→ M2 ──→ M3
 │               │
 └───────────────┼──→ M4 ──→ M5
                 │
                 └──→ M5
```

- M4(댓글/팀)는 M1(회원/인증)과 M3(검색/조회수) 완료 후 시작
- M5(통계)는 M3(조회수 데이터)와 M4(팀 데이터) 이후 시작

## 리스크 항목

| 리스크 | 영향 | 완화 전략 |
|--------|------|----------|
| AI 서비스 모델 미확정 (A-09) | M2 일정 지연 | 인터페이스를 Port로 추상화, mock 구현으로 선행 개발 |
| 인증 방식 미확정 (A-19) | M1 보안 구조 흔들림 | JWT 기본 구현, OAuth 전환 가능 구조 |
| Pinpoint 연동 복잡도 | M3 지연 | Docker Compose 기반 로컬 환경 선구축 후 운영 이관 |
| embedding 대상/chunk 전략 미확정 (A-08) | M2 재작업 | 본문 전체 임베딩으로 시작, chunk 전략은 후속 최적화 |
| 팀 권한 모델 복잡도 | M4 설계 지연 | RBAC 3단계(OWNER/EDITOR/VIEWER)로 시작, 세분화는 후속 |
| 통계 데이터 볼륨 증가 | M5 성능 이슈 | 비동기 집계 + 캐시 전략으로 읽기 성능 확보 |
| 댓글 알림 시스템 범위 확대 | M4 일정 초과 | MVP는 멘션 파싱만, 실시간 알림은 후속 마일스톤 |
