# Stage 01: PRD 모호성 분석

> PRD v2.1 기준 | 분석일: 2026-03-14

## 분석 목적
PRD에서 구현 전 반드시 명확히 해야 할 모호하거나 불완전한 요구사항을 식별한다.

---

## 1. 문서 상태 모델 관련

| # | 모호 항목 | 질문 | 영향도 |
|---|----------|------|--------|
| A-01 | DRAFT → ACTIVE 전환 조건 | 사용자가 명시적으로 전환 버튼을 누르는 것인지, 특정 조건(예: 필수 필드 충족) 시 자동 전환인지? | 높음 |
| A-02 | ACTIVE → DELETED 전환 | ACTIVE 문서를 바로 삭제할 수 있는지, 아니면 DRAFT로 먼저 되돌려야 하는지? PROCESSING 중인 문서의 삭제 처리는? | 높음 |
| A-03 | DRAFT → DELETED 전환 | DRAFT 문서의 삭제가 soft delete인지 hard delete인지? | 중간 |
| A-04 | 상태 복원 가능 여부 | DELETED 문서를 복원할 수 있는지? 복원 시 어떤 상태로 돌아가는지? | 중간 |

## 2. AI 파이프라인 관련

| # | 모호 항목 | 질문 | 영향도 |
|---|----------|------|--------|
| A-05 | AI 파이프라인 재실행 정책 | COMPLETED 이후 문서가 수정되면 AI 파이프라인이 자동 재실행되는지, 사용자가 수동으로 다시 요청해야 하는지? | 높음 |
| A-06 | FAILED 상태 복구 전략 | AI 파이프라인이 FAILED되면 자동 재시도(retry)가 있는지? 재시도 횟수와 간격은? 사용자가 수동 재실행할 수 있는지? | 높음 |
| A-07 | 부분 실패 처리 | summary 성공 → tagger 실패 시, summary 결과는 유지하고 tagger부터 재실행하는지? 전체를 다시 시작하는지? | 높음 |
| A-08 | embedding 대상 범위 | embedding은 본문 전체에 대해 수행하는지, 요약(summary) 결과에 대해 수행하는지? chunk 전략은? | 중간 |
| A-09 | 외부 AI 서비스 | 요약/태깅/임베딩에 사용할 AI 모델/서비스가 정해졌는지? (OpenAI, Claude, 자체 모델 등) | 높음 |
