# DevOps Workspace

인프라 및 운영 자동화 작업은 이 디렉토리를 기준으로 진행합니다.

- Docker, CI/CD, 배포 스크립트, 관측성, 환경 구성
- 필요 시 별도 git worktree로 병렬 브랜치 작업

## DevOps 공동 작업 항목

- `be` lane과 협업해 Pinpoint 모니터링 시스템을 구축합니다.
- Pinpoint 서버/수집기/웹 구성, 배포 방식, 환경 변수, 운영 체크리스트를 관리합니다.

## 기술 문서 규칙

- 구현 중 기술 선택이나 운영 구조 논의가 필요하면 Confluence에 기술 문서를 생성합니다.
- 문서 포맷은 `개요`, `목표`, `설계`를 따릅니다.
- `설계`에는 `데이터 흐름`, `ERD`, `컴포넌트 다이어그램`, `플로우 다이어그램`을 포함합니다.
