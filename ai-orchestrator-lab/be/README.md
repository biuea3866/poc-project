# BE Workspace

백엔드 구현 작업은 이 디렉토리를 기준으로 진행합니다.

- API, 도메인, 이벤트 처리, AI 파이프라인 서버 로직
- 필요 시 별도 git worktree로 병렬 브랜치 작업

## BE 코드 컨벤션

1. 객체지향, 클린 코드, 디자인 패턴 원칙에 기반해 작성합니다.
2. JPA Entity와 도메인 POJO를 분리합니다.
3. 전체 구조는 헥사고날 아키텍처를 따릅니다.

## BE 공동 작업 항목

- `devops` lane과 협업해 Pinpoint 모니터링 연동 지점을 설계합니다.
- 애플리케이션 기동 옵션, 에이전트 주입 방식, 추적 대상 범위를 명확히 정의합니다.

## 기술 문서 규칙

- 구현 중 기술 선택이나 구조 논의가 필요하면 Confluence에 기술 문서를 생성합니다.
- 문서 포맷은 `개요`, `목표`, `설계`를 따릅니다.
- `설계`에는 `데이터 흐름`, `ERD`, `컴포넌트 다이어그램`, `플로우 다이어그램`을 포함합니다.
