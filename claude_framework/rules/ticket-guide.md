# 티켓 작성 가이드

## md 파일 구조

티켓 md 파일은 다음 순서로 작성합니다.

```markdown
# [접두사-NN] 티켓 제목

## 메타
- 사이즈: S
- 의존: A-03~A-09
- Feature Flag: USE_NEW_PROCESSOR (해당 시)

## 작업 내용 (설계 의도)
### 변경 사항
{왜 이 변경이 필요한지, 설계 의도 수준으로 기술}

## 다이어그램
### 처리 흐름
{sequenceDiagram}
### 클래스 의존
{flowchart LR}

## 테스트 케이스
{한 문장}
```

## 포함하지 않는 것

- **AC(인수 조건)**: md에도 Jira에도 넣지 않습니다.
- **생성/수정 파일 목록**: 코드베이스가 기준입니다. 문서에 파일 목록을 관리하지 않습니다.

## 테스트 케이스

한 문장으로 작성합니다. Given/When/Then을 풀어서 쓰지 않습니다.

- "Basic 다운그레이드 시 9개 서비스가 실행된다"
- "만료된 지원자는 상태가 EXPIRED로 변경된다"
- "중복 메시지 수신 시 멱등하게 처리된다"

## 사이즈 정의

| 사이즈 | 기간 | 기준 |
|--------|------|------|
| XS | 0.5일 | 설정 변경, 단순 매핑 |
| S | 1일 | 단일 클래스/테스트 |
| M | 2일 | 여러 클래스 + 통합 테스트 |
| L | 3일+ | 여러 모듈, L 이상이면 세분화 검토 |

## 의존성 표기

- 단일: `의존: A-03`
- 범위: `의존: A-03~A-09`
- 혼합: `의존: A-03~A-09, A-13`
- 의존 없음: 생략합니다.
