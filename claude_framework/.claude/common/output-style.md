# 출력 스타일 가이드

## 문체

- 해요체(~합니다)를 사용합니다.
- 핵심부터 씁니다. 한 문장에 하나의 정보만 담습니다.
- 표와 불릿을 우선 사용합니다. 산문체는 최소화합니다.

## 구체성

- 추상적 표현을 금지합니다. 수치로 표현합니다.
  - "여러 곳" → "3곳"
  - "많은 API" → "12개 API"
  - "느린 응답" → "P95 2.3초"

## 코드 참조 형식

- 메서드 참조: `파일경로#메서드명`
- 라인 참조: `파일경로:42`
- 예시: `ApplicantService.kt#expire`, `V20260401__add_column.sql:15`

## 약어 규칙

- 약어는 첫 등장 시 풀네임을 병기합니다.
  - 예: "TDD(Technical Design Document)"
- 두 번째부터는 약어만 사용합니다.

## 용어 원칙

- 도메인 개념은 영문을 유지합니다: Applicant, Opening, Workspace, Pipeline
- 프로세스/동작은 자연스러운 한국어를 사용합니다: 다운그레이드, 배포, 롤백, 전환
- 상태값은 한글로 표기합니다: 완료, 실패, 부분 실패, 진행 중, 대기
- 기술 용어도 자연스럽게 한글화합니다:
  - fire-and-forget → 결과 미추적
  - 4-phase → 4단계 그룹 병렬
  - blue-green → 블루그린 배포
