# /init — claude_framework 프로젝트 이식

현재 디렉토리에 claude_framework를 설치하고, 옵션에 따라 코드베이스를 스캔해 프로젝트 특화 룰을 추출하거나 하위 저장소를 자동 분류한다.

**입력 플래그**: $ARGUMENTS

## 실행 순서

### Phase 0. 전제 확인

1. 현재 디렉토리가 git repo인지 확인 (`.git/` 존재)
2. 기존 `.claude/` 존재 여부 확인
   - 있고 `--force` 없음 → 중단하고 "이미 초기화됨. --force로 덮어쓰기" 메시지
3. `--stack` 값 해석
   - `auto` (기본): 빌드 파일로 자동 감지 (build.gradle.kts, package.json, pyproject.toml, go.mod)
   - 명시 값: 그대로 사용

### Phase 1. 기본 스캐폴드

플러그인 디렉토리(`${CLAUDE_PLUGIN_DIR:-$HOME/.claude/plugins/claude-framework}`)에서 템플릿 파일 복사:

```
복사 매핑:
  .claude/harness-check.py         → ./.claude/harness-check.py
  .claude/settings.json            → ./.claude/settings.json
  .claude/mcp.json                 → ./.claude/mcp.json (환경변수 치환 후)
  .claude/git-hooks/               → ./.claude/git-hooks/
  common/                  → ./common/
  .analysis/                       → ./.analysis/ (PIPELINE.md들만)
  CLAUDE.md.template               → ./CLAUDE.md (없을 때만)

참조 (복사 안 함 — 플러그인이 제공):
  agents/, skills/, commands/ — 플러그인 네임스페이스 `claude-framework:*`로 자동 노출
```

스택 프리셋 선택:
```
--stack=kotlin → harness-rules.json = 기본 (모든 Kotlin 룰 유지)
--stack=node   → harness-rules.json = node 프리셋 (Kotlin 룰 제거 + TS 룰)
--stack=python → python 프리셋
--stack=go     → go 프리셋
```

`be-repos/ fe-repos/ devops-repos/` 디렉토리는 **비어있는 상태로 생성** (내용 없으면 `.gitkeep`).

### Phase 2. `--scan` (선택) — 코드베이스 컨벤션 스캔

`$ARGUMENTS`에 `--scan` 포함 시:

```
Agent(
  subagent_type="convention-detective",
  model="opus",
  description="코드베이스 컨벤션 추출",
  prompt=<<<
    루트: {현재 디렉토리}
    스택: {감지된 스택}
    임무: skills/codebase-convention-scan/SKILL.md의 6단계 절차 수행.
    출력: 룰 델타 JSON + 승인 요청 표.
  >>>
)
```

에이전트 결과 수신:
1. 제안 요약 표 사용자에게 표시
2. 항목별 승인 (`y/n/skip` 또는 `A` 전체 승인)
3. 승인된 항목을 `harness-rules.json`에 **병합**
   - 백업: `.claude/harness-rules.json.bak-<timestamp>`
   - JSON merge: `forbidden_patterns.rules`에 추가, `layer_dependency.rules`에 추가 등
4. 스모크 테스트:
   ```bash
   python3 .claude/harness-check.py code-pattern <test-input>
   python3 .claude/harness-check.py git-guard <test-input>
   ```
5. 실패 시 자동 롤백

### Phase 3. `--classify-repos` (선택) — 하위 저장소 자동 분류

`$ARGUMENTS`에 `--classify-repos` 포함 시:

```
Agent(
  subagent_type="repo-classifier",
  model="opus",
  description="하위 저장소 BE/FE/DevOps 분류",
  prompt=<<<
    루트: {현재 디렉토리}
    임무: 1-level 서브디렉토리 스캔 + 카테고리 판정.
    출력: 분류 리포트 + 마운트 제안 (심볼릭 링크 스크립트 또는 .mounts.json).
  >>>
)
```

결과 수신:
1. 자동 분류(신뢰도 > 0.7) + 확인 필요 항목 분리해서 표시
2. 사용자에게 마운트 방식 선택 요청:
   - **(A)** 심볼릭 링크 — `be-repos/<name> → ../../{project}`
   - **(B)** `.mounts.json` 매니페스트만 생성 (실제 링크 안 함)
   - **(C)** 건너뛰기 — 수동 마운트
3. 확인 필요 항목은 사용자가 카테고리 선택
4. 선택된 방식으로 실행:
   - (A): 각 분류에 대해 `ln -s` 실행 (기존 링크 있으면 질의)
   - (B): `be-repos/.mounts.json`, `fe-repos/.mounts.json` 등 생성
5. `settings.local.json`의 `permissions.additionalDirectories`에 자동 추가 제안

### Phase 4. CLAUDE.md 초안 생성

Phase 2/3 결과 기반으로 `CLAUDE.md` 자동 채움 (없으면 생성, 있으면 섹션만 추가):

```markdown
# {프로젝트명} — Claude 프레임워크 설정

<!-- CLAUDE_FRAMEWORK:BEGIN (auto-generated by /init) -->

## 스택
- 언어: {감지된 언어}
- 빌드: {빌드 도구}
- 테스트: {감지된 프레임워크}
- 아키텍처: {Hexagonal/Layered/Flat}

## 하위 저장소
| 카테고리 | 레포 |
|---|---|
| BE | {분류된 BE 레포 목록} |
| FE | {분류된 FE 레포 목록} |
| DevOps | {분류된 DevOps 레포 목록} |

## 감지된 컨벤션
{convention-detective 리포트 요약}

<!-- CLAUDE_FRAMEWORK:END -->
```

### Phase 5. 최종 보고

```
## /init 완료

- 스캐폴드: ✓ (.claude/, .analysis/ 생성)
- 스택 감지: {결과}
- 컨벤션 스캔: {수행/건너뜀}
  - 추가된 룰: N개
  - 다운그레이드: M개
- 저장소 분류: {수행/건너뜀}
  - BE: N개 / FE: M개 / DevOps: K개
- CLAUDE.md: {생성/업데이트}

다음 단계:
1. `.claude/mcp.json`의 환경변수 설정 (ATLASSIAN_TOKEN 등)
2. `.claude/settings.local.json` 생성 (개인 퍼미션)
3. Git 훅 연결: `git config core.hooksPath .claude/git-hooks`
4. /analyze-prd 또는 /plan-project로 워크플로우 시작
```

## 옵션 조합 예시

| 명령 | 효과 |
|---|---|
| `/init` | 기본 스캐폴드만 |
| `/init --scan` | 스캐폴드 + 컨벤션 추출 + 룰 주입 |
| `/init --classify-repos` | 스캐폴드 + 하위 저장소 분류 |
| `/init --scan --classify-repos` | 전부 수행 (권장, 신규 멀티 레포) |
| `/init --scan --classify-repos --stack=node` | 스택 명시 |
| `/init --force` | 기존 `.claude/` 덮어쓰기 (주의) |

## 안전장치

- **백업 필수**: 기존 `harness-rules.json`/`CLAUDE.md`는 항상 백업 후 수정
- **사용자 승인**: scan/classify 결과는 항상 승인 후 적용 (`--auto` 없는 한)
- **롤백 가능**: 스모크 테스트 실패 시 자동 롤백
- **멱등성**: 재실행해도 안전 (이미 적용된 룰은 skip)

## 스캔/분류 결과 활용

init 이후 언제든 재실행 가능:
- `/audit-harness` — 현재 코드베이스 대비 룰 위반 감사
- `/scan-conventions` — 컨벤션 재스캔 (별도 커맨드, 향후 추가)
